If order for attacks to succeed remotely, your ISP must provide native support for IPv6. An alternative is to use a tunneling solution from a company such as SixXS or Hurricane Electric. Dynamic, but simple, IPv6 tunnelling can be done with Teredo and Linux miredo daemon

# miredo
# ping6 ipv6.google.com


Other tools may not be compatible with IPv6 at all. Simple tunnels can be constructed in order to allow them to work (though not all functionality can be guaranteed)

Windows:
netsh interface portproxy add v4tov6 listenport=8080 connectport=80 connectaddress=fe80::XXXX

Linux:
socat TCP-LISTEN:8080,reuseaddr,fork TCP6:[fe80:XXXX]:80

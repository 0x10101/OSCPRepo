Browser caches may be forced to reload if the servers are checking for certain headers such as:
If-Modified-Since Server checks to see if local file is newer than the date/timestamp specified in the header; if not newer, server returns 304 Not Modified. Otherwise, server delivers the file.
If-None-Match Similar to the above except with a hash or tag value that the server checks to see if it has been modified.

A possible solution is to remove these headers. The server will detect their absence, cause the client to reload entirely, and then set the headers (which you can continually remove/squash).

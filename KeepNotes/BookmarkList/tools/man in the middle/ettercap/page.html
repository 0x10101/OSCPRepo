Syntax of Etterap is to specify onen or more optoins following the executable, followed by two target designators representing the first and second group of devices that Ettercap should become MITM for.

Common options:
-T Launch in text-only mode
-M [Method:args] Launch and become MitM using specified method arguments (remote, oneway, etc)
-w [pcap] Log all packets to pcap
-z defaut, ettercap scans all hosts in segment. -z supresses this behavior (relying on multicast and broadcast for descovery)
-F [filter file] Execute a specified ettercap filter (generated with etterfilter)
-d Perform DNS resolution (off by default)
-m [Text file] Log messages
-q subdue output (quiet mode)

# ettercap <options> <target1> <target2>

Simple uages:
# ettercap -T -q -M arp:remote /<ip>// /<ip>// Use ettercap to MitM using ARP in both directions (to and from targets)

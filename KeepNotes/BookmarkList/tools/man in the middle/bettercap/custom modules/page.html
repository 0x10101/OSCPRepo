Modules are written in Ruby extending the BetterCap::HTTP::Proxy::Module class. Template:
  • self.on_options define usage optoins, accept command line parameters
  • initialize called when script is loaded
  • on_request called when proxy has an HTTP response; includes request and response data

Other useful extensions:
BetterCap::Logger

Request objects include a headers hash, variables, etc
Response objects are similar including headers, body

private functions indicate that methods and class variables that follow should not be accessible outside of the class itself.
Display value of strings from another string by using: #{variable_name}
has_key?() function tests if a given string representing a hash entry name, returns TRUE or FALSE
sub!('orig','replacement') function is a replacement function

astgt build raw AS-REQ (TGT request) traffic for the specified user and ecryption key
renew will build/parse a raw TGS-REQ/TGS-REP TGT renewal exchange using specified /ticket:X supplied
s4u if a user or computer account has a SPN set in its msds-allowedToDelegateTo field, and an attacker can compromised said user/computer's account hash, that attacker can pretend to be ANY domain user to ANY service on the targeted host
ptt submit a ticket for the current logon session through the LsaCallAuthenticationPackage() API. Reminder, logon session can only have one TGT applied at a time.
purge purge all kerberos tickets from the current logon session
describe takes a /ticket:X value, parses it, and describes the values of the ticket
createnetonly use the CreateProcessWithLogonW() API to create a new hidden process with a SECURITY_LOGON_TYPE of 9 (NewCredentials), the equivalent of runas /netonly
kerberoast replaces SharpRoast project's functionality. All user accounts with SPNs set in the current domain are kerberoasted.
asreproast replaces the ASREPRoast project with BouncyCastle. If domain user does not have Kerberos pre-auth, an AS-REP can be requests for the user and a component of the structure can be cracked offline
dump extract current TGTs and service tickets from memory, if in an elevated context
monitor monitor event log for 4624 logon events and extract any new TGT tickets for the new logon IDs
harvest takes monitor one step further. Monitors 4624 events every /interval:minutes, extracts any new TGT, keeps a cache of any extracted TGTs. Any expiring will be auto renewed

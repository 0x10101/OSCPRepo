Typically implemented when a company wants to enforce a simple authentication and check clients meet minimum security requirements. A dissolvable agent will be downloaded and run on a computer and will remove itself after checks are complete. If checks are passed, access is granted to the network.

Auth
Clientless NAC systems often leverage a captive portal authentication system to validate a user's identity. Once authenticated, the server will grant access to the network.

Validation
ActiveX, Java, or other browser control can launch and perform the system validation.
Captive portals can use a variety of ways to identify and fingerprint the client including MAC address, IP address, User-Agents, a combination of techniques, and more.

Post-Auth
If the server is using out-of-band management, it may use SNMP or an interactive session to grant the end-user access to a second VLAN that has interactive access.


Captive Portals
Lightweight NAC systems use weak authentication controls. Mobile devices often obtain a policy exception due to platform limitations (no Java/Flash, lack of management, etc).
These exceptions in Lightweight NACs create opportunities for bypass/exploitation.

iEEE 802.1x
Switch access ports are "closed" until successful authentication
Three components are generally required:
  • Supplicant or client software
  • The Port Access Entity (PAE)
  • Authentication server (backend RADIUS server)

In wired NAC environments, simple EAP methods are more common. Weak password-based authentication such as EAP-MD5 may be used with certificate-based authentication using EAP/TLS is becoming more popular

Exchange is often (but not always) running with more privileges than necessary. Due to this misconfiguration, it can be possible to relay these credentials for elevated access. This is possible due to Exchange's ability to authenticate to an arbutrary URL over HTTP via the PushSubscription feature.

This was assigned a CVE-2019-0686 and may have been patched/fixed, but still worth checking.
(To address this vulnerability, Microsoft has changed the notifications contract established between EWS clients and Exchange Servers to not allow authenticated notifications to be streamed by the server. Instead, these notifications will be streamed using anonymous authentication mechanisms.)

It is possible to use these credentials to impersonate other users. This is called a reflection attack.
It is possible to use these credentials to grant elevated rights. This is a privilege escalation.

CSRF Protection
Requests include a “X-GWT-Permutation” HTTP header
Provides minimal protection since
  Form submissions cannot set headers
  XmlHttpRequest can not make requests across domains because of Same Origin Policy (SOP)
However, the X-GWT-Permutation value is not unique or tied to a user’s session.
Can be bypassed as long as the X-GWT-Permutation request header can be set.
Since the X-GWT-Permutation is a static value this will not disrupt automated testing

Verb Tampering (ie GETS vs GET, etc) (bypass authentication, more)
The .htaccess specify that the "require valid-user" only applies to "GET" and "POST" requests. So, a "GETS" request is not subject to the "require valid-user".

Why Apache does not reject the invalid "GETS" request ?
Apache does not send a "invalid method" HTTP error message because the resource pointed by the URL is handled by mod_php5 (AddHandler php5-script .php) (or other module), so it assumes that mod_php5 will know what to do with a "GETS" requests. So, it passes the request to mod_php5.
